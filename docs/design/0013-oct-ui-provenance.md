## Where the UI came from

The demo UI in `services/api/static/` was generated by **Claude Design**
(claude.ai/design) in Step 4, from the contract-bound prompt
`docs/design/0013-oct-demo-claude-design-prompt.md`. Cray triggered the
Claude Design → Claude Code handoff; the bundle was fetched from the handoff
URL and unpacked.

**Bundle contents** (Claude Design export):
- `README.md` (handoff instructions), `chats/chat1.md` (the design session —
  intent + iteration), `project/uploads/` (the theme images Cray attached + the
  prompt), and two build forms of the app:
  - `project/OCT Control Tower.html` — a single-file compiled build (all
    CSS/JS inlined).
  - `project/oct/` — the **editable modular source** (HTML shell + `assets/`:
    `theme.css`, `views.css`, `api.js`, `app.js`, `components.js`, `mock.js`,
    `view-map.js`, `view-anomaly.js`, `view-ask.js`, `view-flow.js`).

## What was committed (and why)

We serve the **modular source** (more maintainable than the inlined build):
`project/oct/OCT Control Tower.html` → `services/api/static/index.html` and
`project/oct/assets/*` → `services/api/static/assets/*`. The compiled single-file
build, the chat, the README, and the theme uploads were **not** committed (the
chat + uploads are design-time provenance, captured here by reference).

## How it is wired (Step 5)

- **Served static, same-origin, from FastAPI** (`services/api/main.py`):
  `app.mount("/", StaticFiles(directory=…/static, html=True))`, mounted **last**
  so the JSON API routes take precedence (OQ-4 — one process, one URL, no CORS).
  `/` → `index.html`; `/assets/*` → the bundle; `/meta`, `/objects/{type}`,
  `/recommendations` (+ approve/execute), `/query` → the API routers.
- **The UI is already wired to the live API** by construction: `assets/api.js`
  fetches the relative endpoints first and only falls back to its embedded
  `mock.js` on a non-200 / non-JSON / network failure (flipping the top strip to
  amber **DEGRADED**). Same-origin behind FastAPI it reads **LIVE · SAME-ORIGIN**.
- **Ontology-driven** (AC-template): `api.js`'s `Onto` helpers derive every
  domain word — type names, record labels (`title_key`), enum badges, ref links —
  from `GET /meta`. Nothing about "energy" is hard-coded.
